# Phase 3.4: Test Development Review [PHASE 3.4]

## Subagent Config

- **Primary:** `state.reviewer` (codex-reviewer or claude-reviewer)
- **Criteria:** `prompts/high-stakes/test-review.md`
- **Input:** `.agents/tmp/phases/3.3-test-dev.json`, `.agents/tmp/phases/3.1-test-results.json`
- **Output:** `.agents/tmp/phases/3.4-test-dev-review.json`

## Instructions

Review the tests and CI configuration written by the test-developer agent.

### Process

1. Read the review criteria from `prompts/high-stakes/test-review.md`
2. Read test development results from `.agents/tmp/phases/3.3-test-dev.json`
3. Read original test results from `.agents/tmp/phases/3.1-test-results.json`
4. Review test quality: meaningful assertions, no false positives, good coverage of edge cases
5. Review CI configuration: correct setup, appropriate triggers, coverage enforcement
6. Verify coverage threshold was met (or acceptable reason why not)
7. Write structured JSON result to `.agents/tmp/phases/3.4-test-dev-review.json`

### Output

Write JSON to `.agents/tmp/phases/3.4-test-dev-review.json` with: `status`, `issues[]`, `coverageMet`, `summary`

### If Issues Found

Same retry logic as other review phases: fix, re-review, repeat until approved or max retries.
