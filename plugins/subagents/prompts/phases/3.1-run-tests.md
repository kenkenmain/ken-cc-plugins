# Phase 3.1: Run Tests & Analyze Failures [PHASE 3.1]

## Subagent Config

- **Type:** `state.testRunner` (test-runner or codex-test-runner)
- **Primary Output:** `.agents/tmp/phases/3.1-test-results.json`
- **Secondary Output:** `.agents/tmp/phases/3.2-analysis.md`

## Input Files

- Test commands from config (default: `make lint`, `make test`)

## Output Files

- `.agents/tmp/phases/3.1-test-results.json` — structured test results
- `.agents/tmp/phases/3.2-analysis.md` — failure analysis with fixes applied

The test-runner agent handles both test execution and failure analysis in a single phase. Both output files are produced by the same agent.
