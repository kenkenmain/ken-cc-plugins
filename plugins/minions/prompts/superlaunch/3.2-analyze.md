# Phase 3.2: Analyze Failures [PHASE 3.2]

## Subagent Config

- **Type:** `subagents:failure-analyzer` (reads from `state.failureAnalyzer`)
- **Output:** `.agents/tmp/phases/3.2-analysis.md`

## Input Files

- `.agents/tmp/phases/3.1-test-results.json`

## Instructions

Analyze test and lint failures from Phase 3.1:

- Identify root causes for each failure
- Apply fixes directly to the codebase
- Re-run failing tests to verify fixes work
- If a failure cannot be fixed, document the reason

Write a structured analysis to `.agents/tmp/phases/3.2-analysis.md` with:
- List of failures analyzed
- Fixes applied (file, change description)
- Remaining unfixed failures (if any) with reasons
